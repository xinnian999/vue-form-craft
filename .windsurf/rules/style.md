## 样式规范

1. **统一引入基础样式**：开发任意组件且涉及样式时，必须先引入 `@/style.scss`，并优先复用文件中已有的主题或颜色变量，保证开源项目的整体视觉一致性。
2. **命名空间配套使用**：当模板里通过 `ns` 生成命名空间类名时，样式写法也要配合 `@include ns('xxx') { ... }`，避免命名空间与样式脱节。
3. **集中管理变量**：为了后续多 UI 库适配，若需要新增颜色或样式变量，请在 `@/style.scss` 中扩展并可引用 Element Plus 的 `var(--el-xxx)`；不要在组件内部直接读取这些变量。
